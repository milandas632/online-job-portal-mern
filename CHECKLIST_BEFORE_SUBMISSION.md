# Checklist Before Qollabb Submission

- [ ] Create a public GitHub repository.
- [ ] Upload every project file except `.env` and `node_modules`.
- [ ] Create a MongoDB Atlas database user and copy the connection string.
- [ ] Deploy the repository through Render using `render.yaml`.
- [ ] Add `MONGODB_URI` in Render environment variables.
- [ ] Run `npm run seed` once in the Render shell.
- [ ] Test candidate, recruiter, and administrator accounts.
- [ ] Add the live Render URL near the top of `README.md`.
- [ ] Capture real website screenshots for the report or presentation.
- [ ] Paste the GitHub repository or live-demo URL into Qollabb.
- [ ] Never upload `server/.env` or any password/secret.
